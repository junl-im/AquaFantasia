# AquaFantasia v2.0.36

고퀄리티 SD 해양 판타지 RPG 방향으로 전환 중인 PixiJS 기반 모바일 세로형 웹게임입니다. 현재 목표는 메뉴형 낚시 게임에서 벗어나, 플레이어가 직접 루미나 베이 마을을 돌아다니고 NPC, 건물, 항구, 건설 시스템을 자연스럽게 이용하는 첫 마을 RPG 기반을 완성하는 것입니다.

## 현재 방향

- 장르: SD 해양 판타지 RPG
- 화면: 모바일 세로모드 1080 x 1920 기준
- 렌더링: PixiJS 8 기반 2.5D 마을 월드
- 조작: 좌측 가상 조이스틱, 터치 이동, 우측 간소 메뉴, 캐릭터 시점 확대/축소
- 배포: GitHub Pages 가능 구조
- 저장: 로컬 저장 우선, Firebase 연동 준비 구조 유지

## v2.0.36 변경사항

- 낚시게임의 핵심 플레이 UI를 다시 안전 레이아웃으로 재구성했습니다.
  - 릴 감기 패널을 좌우 `left/right` 늘림 방식이 아니라 `left: 50vw` 중심 고정 방식으로 바꿔, 창이 비틀리거나 왼쪽/오른쪽으로 밀리지 않게 했습니다.
  - 패널 폭은 `min(388px, 100vw - 24px)`로 제한해서 모바일 화면 밖으로 절대 나가지 않게 했습니다.
  - 기존 왜곡되던 게이지 이미지 의존도를 줄이고, CSS 기반 세로 장력 게이지/가로 장력 트랙/안전 진행바/서지 미터/릴 버튼을 항상 보이게 했습니다.
  - `v205-horizontal-gauge`, `v205-vertical-gauge`, `v205-resistance-art` 이미지는 이번 낚시 플레이 UI에서는 숨겨서 틀어짐을 줄였습니다.
- 낚시 성공 결과창이 우측 화면 밖으로 밀리는 문제를 수정했습니다.
  - 결과창을 `stageHost` 내부 절대 위치가 아니라 `.fishing-screen` 기준 fixed 중앙 팝업으로 붙이도록 변경했습니다.
  - 결과창은 `left: 50vw`, `top: 50dvh`, `max-width: 100vw - 28px`로 고정해 화면 밖으로 나가지 않습니다.
  - 결과창 내부 물고기 이미지, 보상 문구, 버튼 높이를 줄여 작은 화면에서도 잘리지 않게 했습니다.
- 낚시 화면 전체 밀림 방지를 다시 강화했습니다.
  - fishing root, stage, Pixi canvas, fallback scene, water overlay를 모두 `100vw x 100dvh` fixed layer로 재고정했습니다.
  - 낚시 시작 버튼은 중앙 하단 도크 위 안전 위치에 고정했습니다.
  - 큰 바다 장식/게이지 프레임 이미지는 낚시 조작 UI를 방해하지 않도록 숨겼습니다.
- 우측 하단 메뉴 도크를 v2.0.36 기준으로 다시 고정했습니다.
  - 홈/낚시/메뉴 모두 `v2036-identical-dock-nav`를 사용합니다.
  - 하단 safe-area 여백, 흰색 글씨, 투명 테두리, `[마을] / [가방][퀘스트][지도]` 구조를 유지합니다.
  - 캐릭터 정보창이 열리면 `v2036-character-panel-open` 상태로 도크가 숨겨집니다.
- 우측 상단 조작바는 기존 좁은 간격을 유지하고, 테두리/그림자는 더 약하게 보정했습니다.
- `check-v2036-fishing-gauge-result-safe-layout.mjs` 검증을 추가했습니다.
  - 낚시 릴 패널, 결과창, 중앙 fixed 배치, 도크, 캐릭터 정보창 도크 숨김, registry 오염 여부를 검사합니다.
- `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge를 `2.0.36`으로 동기화했습니다.
- README.md만 수정했고 별도 `*_NOTES.md`는 생성하지 않았습니다.

## v2.0.33 변경사항

- 낚시게임 플레이 가능성을 최우선으로 다시 고정했습니다.
  - 낚시 화면 root/stage/pixi/fallback 레이어를 다시 `100vw x 100dvh` 안전 영역으로 묶어 왼쪽으로 밀리는 문제를 더 강하게 차단했습니다.
  - 릴 감기 패널은 `v2033-reel-panel` 기준으로 도크 위 중앙에 고정하고, 가로 게이지/세로 게이지/장력 트랙/안전 진행바/서지 미터/릴 버튼이 모두 보이도록 display/width/height를 재고정했습니다.
  - 낚시 시작 버튼은 더 작게 줄이고 중앙 하단 안전 영역에 고정했습니다.
  - 최근 획득 카드가 상단 HUD와 겹치는 문제를 막기 위해 낚시 화면에서는 숨김 처리했습니다.
  - 바다 배경의 큰 장식 오브젝트는 계속 제거하고 핵심 게임 UI를 우선 표시합니다.
- 우측 하단 메뉴바를 홈/낚시/메뉴 공통 `v2033-identical-dock-nav`로 다시 잠갔습니다.
  - 버튼 테두리는 투명 처리하고, 설명 글씨는 흰색을 유지합니다.
  - 하단 safe-area 여백을 확보해 도크가 바닥에 파묻히거나 잘리지 않도록 했습니다.
- 우측 상단 `+ / - / 원점 / 건설 / 상점 / 출항` 조작바를 다시 줄였습니다.
  - 버튼 사이 간격을 최대한 좁히되 붙지 않게 조정했습니다.
  - 테두리 색은 투명에 가깝게 되돌리고 기존 아쿠아 톤을 유지합니다.
- 게임종료/뒤로가기 확인 팝업을 수정했습니다.
  - 제목과 본문 글씨를 흰색으로 고정했습니다.
  - 긴 문구가 카드 밖으로 넘치지 않도록 width, line-height, overflow-wrap, 버튼 ellipsis를 추가했습니다.
- HUD와 캐릭터 정보창을 다시 점검했습니다.
  - HUD 프로필 칩과 캐릭터 정보창에서 `정보` 배지/라벨 느낌의 pseudo element를 계속 숨깁니다.
  - 캐릭터 정보창이 열리면 우측 하단 메뉴 도크는 숨김 상태가 됩니다.

  - 좌/우/상/하 및 7시/11시는 기존 방향 규칙을 유지합니다.
- `check-v2033-fishing-ui-direction-dialog-audit.mjs` 검증을 추가했습니다.
  - 낚시 UI, 릴 게이지, 도크 투명 테두리, 게임종료 팝업 흰 글씨/넘침 방지, 캐릭터 대각선 텍스처 보정, registry 오염 여부를 검사합니다.

## v2.0.30 변경사항

- 지적된 화면별 문제를 원인 기준으로 다시 점검했습니다.
  - 마을 로딩 화면에서 우측 하단 메뉴바가 같이 보이던 문제는 `mountBottomNav()` 호출 시점이 너무 빨랐던 것이 원인이었습니다.
  - v2.0.30부터는 마을 Pixi 월드 초기화가 끝나고 `.v2027-village-ready`가 붙은 뒤에만 메뉴바를 붙입니다.
  - sibling nav까지 숨기는 `v2030` CSS guard도 추가해 로딩 중 도크/우측 조작바가 먼저 보이지 않도록 막았습니다.
- 홈 HUD와 우측 상단 조작바를 다시 줄였습니다.
  - `+ / - / 원점 / 건설 / 상점 / 출항` 버튼은 HUD 우측에 붙는 작고 촘촘한 세로바로 재정렬했습니다.
  - `+ / - / 원점 / 상점` 버튼도 `건설 / 출항` 버튼과 같은 크기·테두리·간격 기준을 쓰도록 고정했습니다.
  - 팝업이나 건설 트레이가 열리면 우측 조작바는 뒤로 밀리고 팝업이 최상위가 됩니다.
- HUD와 내 정보창을 다시 정리했습니다.
  - 내 정보창의 `CAPTAIN PROFILE`, 설명문, 정보 라벨 느낌의 장식을 숨겼습니다.
  - 검은 음영, 두꺼운 테두리, 불필요한 장식 pseudo element를 제거하고 밝은 아쿠아 카드 톤으로 통일했습니다.
- 메시지 토스트를 수정했습니다.
  - 표시 시간을 2초로 줄였습니다.
  - HUD와 겹치지 않도록 더 아래로 내리고, 길이는 HUD보다 과하게 길지 않은 짧은 배너로 다시 줄였습니다.
- 낚시터 화면을 다시 고정했습니다.
  - 낚시 stage를 `v2030-fishing-stage-reset` 기준으로 전체 화면 중앙에 고정해 왼쪽으로 밀리는 문제를 막았습니다.
  - 바다 배경에 크게 튀어나오던 장식 오브젝트는 숨기거나 92px 이하·낮은 opacity로 줄였습니다.
  - 낚시 시작 버튼은 중앙 하단 안전 영역에 고정하고 왼쪽 구석으로 밀리지 않도록 `left: 50% + translateX(-50%)`로 재정렬했습니다.
  - 릴 감기 패널, 가로 게이지, 세로 게이지, 장력 트랙을 다시 보이도록 display/width/height guard를 추가했습니다.
  - 최근 획득 물고기 창은 상단 HUD와 겹치지 않도록 하단 도크 위의 작은 정보 카드로 이동하고, 릴링/결과 중에는 숨깁니다.
  - `물었다!`, `릴링!` 같은 액션 팝업은 흰색 글씨와 어두운 그림자로 통일했습니다.
- 우측 하단 메뉴바를 다시 고정했습니다.
  - 마을/낚시/메뉴 화면 모두 같은 `[마을] / [가방][퀘스트][지도]` 도크 구조를 유지합니다.
  - 아이콘 설명 글씨는 흰색으로 유지합니다.
  - 구버전 하단바가 밑으로 파묻히거나 다시 보이는 문제를 막기 위해 v2030 최종 guard를 추가했습니다.
- 가방/퀘스트/지도/상점 화면을 다시 다듬었습니다.
  - 과한 프레임이 본문을 밀어내지 않도록 runtime content 영역의 padding, opacity, visibility를 다시 고정했습니다.

  - 작업 컨테이너에서 `registry.npmjs.org` DNS 조회가 `EAI_AGAIN`으로 실패하면 패키지 코드 문제가 아니라 외부 npm registry 네트워크/DNS 접근 제한으로 판단
  - GitHub Actions에서는 public npm registry 강제, package-lock 내부 registry 오염 검사, 설치 로그 금지 문자열 검사를 계속 유지
  - `ci:registry:check` 스크립트와 workflow 진단 단계를 추가해, 실제 CI에서 네트워크 문제인지 lockfile 오염인지 구분하기 쉽게 함

- 낚시터 찌/게이지 안정화
  - Pixi 낚시 화면에서 캐스팅 전 찌가 미리 떠 있는 문제를 막기 위해 `idle` 상태에서는 bobber를 숨기고 캐스팅 시작 때만 표시
  - HTML fallback 낚시 화면도 `idle/success/fail` 상태에서는 찌를 숨기고 `casting/waiting/bite/reeling` 상태에서만 보이도록 phase 클래스를 추가
  - 낚시 게이지 패널을 화면 하단 안전 영역 안으로 고정하고, 작은 화면에서도 horizontal/vertical gauge, hold button, progress bar가 밖으로 밀리지 않도록 크기 제한
  - 릴링 중에는 최근 포획 패널을 자동으로 숨겨 게이지와 겹치지 않게 함
  - fallback 릴링 루프의 `const safe` 중복 선언을 제거해 TypeScript 빌드 실패 후보를 수정

- 낚시터 에셋 추가 적용
  - 기존 보유 에셋 중 물결 링, 해안 거품, 물고기 그림자, 작은 물튀김을 낚시 stage에 배치
  - 추가 낚시 ambient 에셋을 service worker precache에 연결해 오프라인 캐시 누락을 줄임

- 버전과 검증 갱신
  - `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge를 `2.0.19`로 통일
  - `npm run validate`는 v2.0.19 낚시터 안정화, registry 진단, README-only 규칙, 8방향 반전 방지를 함께 확인
  - 루트 문서는 계속 `README.md` 하나만 유지하고 별도 `*_NOTES.md`는 생성하지 않음

## 조작 방식

- 좌측 조이스틱: 자유 이동
- 마을 타일 터치: 해당 위치로 이동
- NPC 근처 터치: 이동 후 대화창 표시
- 여관, 어시장, 낚시 길드, 항구 사무소 터치: 건물 내부 팝업 표시
- 우측 `+`, `-`: 플레이어 중심 확대/축소
- 우측 `◎`: 플레이어 위치로 카메라 복귀
- 우측 `건설`: 건설 팝업 열기/닫기
- 우측 `출항`: 낚시 화면 이동
- 우측 하단 메뉴: 가방, 퀘스트, 지도, 마을 화면 진입용 2x2 간소 메뉴

## 에셋 반영 원칙

- 새 에셋은 한 번에 무리하게 전부 교체하지 않고, 화면별로 안정적으로 반영합니다.
- 우선순위는 다음과 같습니다.
  1. 캐릭터/NPC 스프라이트 안정화
  2. 대화창 + NPC 초상화
  3. 상단 HUD/우측 하단 메뉴/팝업 UI 프레임
  4. 건물 내부 진입 화면
  5. 월드맵 UI
  6. 낚시 미니게임 UI/VFX
- 이미지 파일명은 기능을 알 수 있게 유지합니다.
- 루트에는 에셋 설명용 별도 MD를 만들지 않습니다. 필요한 설명은 이 README에만 적습니다.
