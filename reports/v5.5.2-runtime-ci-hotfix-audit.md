# v5.5.2 Runtime & CI Hotfix Audit

## 발견한 문제

1. GitHub Actions 경고
   - 기존 workflow는 `actions/checkout@v4`, `actions/setup-node@v4`를 사용했습니다.
   - 2026년 기준 GitHub Actions JavaScript action 런타임이 Node 24로 이동 중이라 Node 20 deprecation 경고가 발생합니다.

2. Validate 실패 가능성
   - 일부 정적 검증 스크립트가 `APP_VERSION = '5.5.0'` 또는 `aqua-fantasia-v5.5.0-mobile-feel-20260612`만 엄격히 허용했습니다.
   - v5.5.1/v5.5.2처럼 핫픽스 버전을 올리면 실제 파일은 정상이어도 `validate`가 실패할 수 있었습니다.

3. 게임 실행 오류 가능성
   - v5.5 계열은 과거 단일 HTML inline 이벤트와 v5.3+ 모듈 런타임이 함께 남아 있어 CAST/BITE/REEL이 중복 실행될 수 있습니다.
   - PWA Service Worker가 오래된 런타임 모듈을 반환하면 GitHub Pages 배포 직후에도 브라우저는 이전 코드를 실행할 수 있습니다.

## 수정 내용

- v5.5.1 런타임 오류 핫픽스 유지.
- v5.5.2 런타임 가드 추가.
- Service Worker 캐시 버전 갱신.
- 모든 런타임 모듈 URL `?v=5.5.2` 적용.
- GitHub Actions official action을 v6로 갱신.
- Node 24 검증 워크플로 적용.
- 정적 검증 스크립트를 v5.5.x 핫픽스 흐름에 맞게 완화.
