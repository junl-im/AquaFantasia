# v5.5.4 Stack Guard Audit

## Problem

After v5.5.2c, validation passed, but two issues remained:

1. Two workflow files could both run on push if the previous static validation workflow remained active.
2. The browser could show `게임 초기화 중 오류... Maximum call stack size exceeded` when cached/runtime layers wrapped the same global fishing handlers or an optional runtime init stage failed during boot.

## Fix

- `.github/workflows/pages.yml` is the only automatic push workflow.
- `.github/workflows/aqua-static-validate.yml` is manual-only.
- `v554-stack-guard.js` provides recursion detection, cache recovery UI, and boot-stage isolation.
- `index.html` now runs optional visual/runtime boot steps through `runAquaBootStage()`.
- v5.5.1 and v4.9 global handler proxies now include re-entry guards.

## Verification

The following commands passed in the patched project:

```bash
npm run validate
npm run typecheck
npm run runtime:check
npm run renderer:check
npm run audit
npm run atlas:check
npm run clean:report
```
