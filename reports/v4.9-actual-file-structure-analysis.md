## 발견 사항

### 불필요하거나 배포에 비효율적인 파일

1. `.git` 전체가 업로드 ZIP에 포함되어 있습니다. GitHub Desktop 작업용으로는 괜찮지만, 배포/전달용 ZIP에는 제외하는 것이 좋습니다.
2. `PATCH_NOTES_v1.2.md`부터 `PATCH_NOTES_v5.4.md`까지 과거 문서가 누적되어 있습니다. 개발 이력 보관용으로는 좋지만 Pages 배포에는 필요하지 않습니다.
3. `V3_*`, `V4_*`, `V5_*` 체크리스트가 다수 남아 있습니다. 최신 검증에 필요한 일부만 루트에 유지하고 나머지는 `docs/archive/`로 이동하는 편이 좋습니다.
4. `tools/script0.js`는 0바이트입니다.
5. `tools/script2.js`는 125 KB 이상이며, 현재 GitHub Actions 검증 경로에서는 직접 사용되지 않습니다.
6. 한글 파일명이 ZIP에서 `#U...` 형태로 깨진 `assets/ui-kit/panels` 파일이 있습니다. 현재 런타임은 `panel_1.png` 중심으로 참조하므로 신규 UI는 안전 파일명을 사용해야 합니다.

### 중복 코드 / 버전 불일치

1. README는 v4.9, package는 v5.4, manifest는 v5.1, service worker는 v5.4로 표기가 섞여 있었습니다.
2. `index.html` 안에 v3~v5 세대의 CSS/JS가 누적되어 parse 비용과 UI 충돌 가능성이 있습니다.
3. `src/runtime/v52/v53`의 캐주얼 낚시 시스템과 `index.html` 내부 낚시 시스템이 병존합니다. 현재는 이벤트를 느슨하게 연결하는 전환기 구조입니다.

### UI 겹침

1. 낚시 화면에서 하단 네비게이션, 원핸드 FAB, 몰입 버튼, 구형 도크가 액션 영역과 겹칠 수 있습니다.
2. `fishing-stage-guide`, `fishing-status`, v41/v44/v53 계열 안내 UI가 작은 화면에서 수직 공간을 경쟁합니다.
3. 릴 단계에서 기존 릴 패널과 화면 내부 안내가 동시에 보여 입력 우선순위가 흐려질 수 있습니다.

### 성능 병목 / 모바일 렉 가능성

1. `index.html` 670 KB 이상, 약 1만 줄로 초기 파싱 비용이 큽니다.
2. 낚시 화면에 Canvas/DOM/SVG 애니메이션 레이어가 여러 세대 중첩됩니다.
3. SVG 아트가 296개 이상 존재합니다. Service Worker precache는 제한되어 있지만 저장소/전송 ZIP 크기는 계속 증가합니다.
4. 저사양 장치에서 애니메이션과 DPR 캡이 동시에 관리되지 않으면 배터리/발열/프레임 저하가 발생할 수 있습니다.

### Service Worker 캐시 문제

1. 기존 캐시 버전 `aqua-fantasia-v5.4.0`은 새 패치 파일을 강제 반영하기에 약했습니다.
2. `index.html`, 런타임 JS, manifest가 섞여 캐시되면 GitHub Pages에서 오래된 UI가 남을 수 있습니다.
3. 이번 패치에서 버전을 `aqua-fantasia-v5.4.0-v49-action-20260612`로 올리고 신규 런타임을 precache했습니다.

### 낚시 조작감

1. 기존은 릴 버튼 연타 중심이라 모바일에서 손맛은 있으나 피로도가 높았습니다.
2. 이번 패치에서 화면 내부 `꾹 눌러 릴 감기` 패드와 기존 `reelAction()`을 연결해 누르고 떼는 감각을 보강했습니다.
3. 안전지대 30~70 안내를 시각적으로 강화했습니다.
