# AquaFantasia v2.0.11

고퀄리티 SD 해양 판타지 RPG 방향으로 전환 중인 PixiJS 기반 모바일 세로형 웹게임입니다. 현재 목표는 메뉴형 낚시 게임에서 벗어나, 플레이어가 직접 루미나 베이 마을을 돌아다니고 NPC, 건물, 항구, 건설 시스템을 자연스럽게 이용하는 첫 마을 RPG 기반을 완성하는 것입니다.

## 현재 방향

- 장르: SD 해양 판타지 RPG
- 화면: 모바일 세로모드 1080 x 1920 기준
- 렌더링: PixiJS 8 기반 2.5D 마을 월드
- 조작: 좌측 가상 조이스틱, 터치 이동, 우측 간소 메뉴, 캐릭터 시점 확대/축소
- 배포: GitHub Pages 가능 구조
- 저장: 로컬 저장 우선, Firebase 연동 준비 구조 유지

## v2.0.11 변경사항

- ㅢ형 우측 최하단 메뉴 도크 적용
  - 메뉴 배치를 `가방 / 퀘스트 / 지도` 하단 3개 + `마을` 상단 1개 구조로 변경
  - 지도 버튼 바로 위가 마을 버튼이 되도록 우측 기준 ㅢ형 그리드로 고정
  - 메뉴바를 둘러싸던 큰 테두리와 프레임은 완전히 제거
  - 아이콘 크기는 과하게 크지 않게 적정 크기로 재조정

- 전 화면 메뉴 도크 겹침 방지
  - 가방, 퀘스트, 지도, 마을 등 모든 메뉴 화면에 우측 하단 도크 안전 여백 적용
  - 화면 속 UI가 메뉴 도크 아래로 깔리지 않도록 스크롤 하단 여백과 콘텐츠 폭을 재정리
  - 낚시와 로그인 화면에서는 도크가 보이지 않도록 유지

- 건설 시스템 개선
  - 건설 팝업에서 원하는 건물/오브젝트를 클릭하면 팝업이 닫히고 설치 모드로 진입
  - 선택한 건물이 반투명 프리뷰로 따라다니도록 개선
  - 설치 가능 타일은 초록색, 설치 불가 타일은 빨간색으로 표시
  - 타일을 터치하면 즉시 설치하고, 건물 설치 후에는 일반 모드로 복귀

- 이름표 레이어 개선
  - 건물 이름표를 건물/오브젝트 스프라이트보다 위 레이어로 분리
  - 캐릭터 이름표와 건물 이름표가 캐릭터 좌우 반전의 영향을 받지 않도록 유지
  - 마을 안의 시설 이름이 소품에 가려지지 않도록 보정

- 캐릭터 방향 재점검
  - SD 캐릭터 기본 방향을 기준으로 좌우 반전 방향을 다시 점검
  - 이동 방향과 캐릭터가 바라보는 방향이 일치하도록 조정
  - 기존 제공 8방향 캐릭터 시트는 유지하며, 현재 화면에서는 안정적인 좌우/전면 기반 표시를 우선 사용

- 버전과 검증 갱신
  - `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge를 `2.0.11`로 통일
  - `validate`는 v2.0.11 메뉴 도크, 건설 프리뷰, 이름표 레이어, 캐릭터 방향 기준을 확인
  - 변경사항은 계속 `README.md`에만 기록
  - 루트 문서는 계속 `README.md` 하나만 유지

## 조작 방식

- 좌측 조이스틱: 자유 이동
- 마을 타일 터치: 해당 위치로 이동
- NPC 근처 터치: 이동 후 대화창 표시
- 여관, 어시장, 낚시 길드, 항구 사무소 터치: 건물 내부 팝업 표시
- 우측 `+`, `-`: 플레이어 중심 확대/축소
- 우측 `◎`: 플레이어 위치로 카메라 복귀
- 우측 `건설`: 건설 팝업 열기/닫기
- 우측 `출항`: 낚시 화면 이동
- 우측 하단 메뉴: 가방, 퀘스트, 지도, 마을 화면 진입용 2x2 간소 메뉴

## 에셋 반영 원칙

- 새 에셋은 한 번에 무리하게 전부 교체하지 않고, 화면별로 안정적으로 반영합니다.
- 우선순위는 다음과 같습니다.
  1. 캐릭터/NPC 스프라이트 안정화
  2. 대화창 + NPC 초상화
  3. 상단 HUD/우측 하단 메뉴/팝업 UI 프레임
  4. 건물 내부 진입 화면
  5. 월드맵 UI
  6. 낚시 미니게임 UI/VFX
- 이미지 파일명은 기능을 알 수 있게 유지합니다.
- 루트에는 에셋 설명용 별도 MD를 만들지 않습니다. 필요한 설명은 이 README에만 적습니다.

## 문서 정책

- 루트 문서는 `README.md` 하나만 사용합니다.
- `PATCH_NOTES.md`, `*_NOTES.md`, `FINAL_CONSOLIDATED.md`, `CLEAN_REPLACE_GUIDE.md` 같은 별도 패치 문서는 만들지 않습니다.
- 변경 내역은 이 README의 버전 섹션에만 추가합니다.

## 적용 방법

전체 덮어쓰기 패치 사용을 권장합니다.

```bash
npm install
npm run dev
```

배포 전 검증:

```bash
npm run typecheck
npm run build
npm run validate
```

## 검증 명령

```bash
npm run validate
npm run typecheck
npm run build
```

현재 `validate`는 다음을 확인합니다.

- 루트 문서가 `README.md` 하나뿐인지
- 버전이 `2.0.x` 체계를 따르는지
- `package.json`, `APP_VERSION`, service worker cache, offline page가 같은 버전을 쓰는지
- 조이스틱, 건설 팝업, SD 캐릭터, 간소 메뉴 에셋이 연결되어 있는지
- v2.0.9 오브젝트 에셋, 건설 버튼 이미지, 우측 하단 2x2 메뉴, 기존 화면 안전 레이아웃이 연결되어 있는지
