# v6.5.1 Portrait Lock Audit

## Root cause
- v6.3~v6.5 runtime preserved the device orientation at launch.
- That allowed landscape locks when the browser reported a wide viewport.

## Fix
- Set manifest orientation to `portrait-primary`.
- Replace runtime orientation policy with `portrait-primary` first, `portrait` fallback.
- Remove landscape-specific UI compaction that made the app feel rotated.
- Keep no-popup behavior when the browser denies orientation/fullscreen lock.

## Compatibility
- Android Chrome PWA: portrait lock should apply through manifest and runtime lock.
- iOS Safari/PWA: manifest behavior depends on platform policy; CSS keeps portrait shell if physical viewport changes.
- Desktop/browser preview: centered portrait shell remains usable.
