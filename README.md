# AquaFantasia v2.0.96

## v2.0.96 변경사항

- UI 안정화 전용 패치입니다. 새 기능 확장보다 실제 화면 복구를 우선했습니다.
- `v2096` 런타임 셸을 추가해 마을 HUD, 메뉴 페이지, 개척, 닫기 버튼, 하단 메뉴바, 낚시 화면을 한 기준으로 다시 묶었습니다.
- 마을 HUD 길이를 줄이고 우측 조작 버튼이 HUD 옆에 붙도록 재배치했습니다.
- 사라졌던 하단 한줄형 메뉴바를 `v2096-bottom-nav`로 새로 만들었습니다. 구형 도크가 아니라 가방/퀘스트/지도/상점/장비/도감/출항/마을이 한 줄로 들어가는 새 구조입니다.
- 상점 페이지 빈 화면 원인이었던 `runtime-content` 호환 구조를 유지하고, 메뉴 본문을 `v2096-menu-content` 단일 스크롤 카드로 다시 고정했습니다.
- 개척 버튼과 개척 본문을 `v2096-expedition-*` 상태로 분리해 버튼 클릭 시 중앙 아쿠아 카드가 열리도록 다시 고정했습니다.
- 모든 일반 메뉴/팝업은 캐릭터 정보창 계열의 밝은 아쿠아 카드, 흐릿한 바깥 배경, 우측 상단 원형 `X` 기준을 유지합니다.
- 시작 화면의 `이 기기에 로그인 유지` 버튼 글씨와 체크 표시를 덜 눈 아프게 조정하고, 체크 상태가 더 분명히 보이도록 했습니다.
- 낚시 화면에 `v2096-fishing-screen` 안정화 셸을 추가해 HUD/닫기/캐스팅/릴 콘솔/하단 메뉴 배치를 다시 잡았습니다.
- 마을 타일 터치 기준을 사용자가 누르는 좌측 아래 발판 쪽으로 약간 보정했습니다.
- `v2.0.96` 전용 검증 스크립트 `tools/check-v2096-ui-restored-runtime-shell.mjs`를 추가했습니다.
- `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge를 `2.0.96`으로 동기화했습니다.

## 검증

- `npm run validate` 기준으로 패키지 구조, 버전 동기화, 클린 UI 셸, 하단 메뉴바, 개척/상점/낚시/터치 보정 가드를 확인합니다.
- 현재 작업 환경에서는 npm registry DNS 문제로 `npm ci`, `typecheck`, `build`가 외부 의존성 미설치 상태에서 실패할 수 있습니다.
