# v6.0.0 Interaction Balance Audit

## 발견한 문제
1. 기존 토스트는 `pointer-events:none` 성격이 강해 사용자가 즉시 닫기 어려웠습니다.
2. 미션/보상 알림도 단순 문구로만 표시되어 다음 행동으로 연결되지 않았습니다.
3. 난이도는 후반 장비 강화 이후 너무 쉽게 안정화될 수 있었습니다.
4. v5.7 배경은 좋아졌지만 수면 질감/반짝임이 화면별로 더 필요했습니다.

## 적용한 조치
- `v60-interaction-balance.js`에서 알림을 `data-aqua-notice` 대상으로 장식하고 탭/스와이프 닫기를 추가했습니다.
- 텍스트 패턴 기반으로 미션/보상/출석 알림은 `quests`, 장비/상점/강화 알림은 `equipment`, 가방/판매/도감 알림은 `inventory`, 낚시 알림은 `fishing`으로 라우팅합니다.
- `alert()`는 브라우저 기본 팝업 대신 모바일 카드형 알림으로 표시합니다.
- 낚시 밸런스는 입질 창, 릴 타이밍 폭, 커서 속도, 장력 상승량, 릴 획득량을 소폭 조정했습니다.
- `v60_water_depth_overlay.webp`, `v60_caustic_sparkle_overlay.webp`를 추가해 배경 수면 레이어를 보강했습니다.

## 리스크 관리
- 실제 모달/종료 확인창은 무분별하게 닫히지 않도록 토스트/알림 계열만 대상으로 제한했습니다.
- `prefers-reduced-motion`, `perf-lite`, Save-Data 환경에서는 수면 애니메이션 강도를 줄입니다.
- 기존 v5.9 도감/라우터 검증은 v6.0 저장 버전과 함께 통과하도록 호환 처리했습니다.
