# AquaFantasia v2.0.81

## v2.0.81 변경사항

- 누적 CSS 오버라이드 때문에 HUD 내부 배치, 메뉴 스크롤, 버튼/아이콘 잔상, 개척바 계층이 서로 꼬이던 문제를 `v2081` 최종 관리 레이어로 정리했습니다.
- 열린 메뉴 페이지에 캐릭터 정보 팝업 계열의 공용 아쿠아 흐림 오버레이를 추가했습니다. 낚시터는 풀화면 플레이 화면이므로 제외합니다.
- 가방/지도/퀘스트/상점/장비/도감/랭킹 메뉴의 `runtime-hud`와 `runtime-content` 구조를 다시 고정해 상단 정보와 본문 카드가 겹치지 않도록 했습니다.
- 본문보다 카드 칸이 좁게 잡히는 문제를 줄이기 위해 모든 메뉴 카드/목록/요약판의 `min-width`, `max-width`, overflow, line-clamp를 재정규화했습니다.
- 드래그 스크롤 타깃을 `v2081-scroll-root`, 개척 본문, 카드 흐름, 메뉴 본문 하위 요소까지 확장해 일부 페이지에서 스크롤이 먹지 않는 현상을 완화했습니다.
- 버튼 하단에 줄이 보이거나 아이콘 뒤에 다른 조각이 살짝 보이는 원인을 오래된 sprite/pseudo 버튼 스킨 잔상으로 보고, 메뉴/팝업 버튼의 `::before`/`::after`와 배경 이미지를 최종 레이어에서 제거했습니다.
- 마을 HUD는 외곽 아쿠아 카드 테두리를 유지하면서 내부 프로필/지갑/마일스톤 배치를 2열 지갑 구조로 재정렬했습니다.
- 개척바는 HUD와 너무 붙지 않게 간격을 다시 벌리고 길이를 줄였으며, 개척 창이 열리면 바 자체는 숨기고 창 본문만 보이도록 수정했습니다.
- 캐릭터/건물/건설 팝업과 메뉴 페이지가 열릴 때 배경 사이드가 흐릿해지는 공용 아쿠아 카드 모달 효과를 유지하도록 정리했습니다.
- `v2.0.81` 전용 검증 스크립트 `check-v2081-aqua-ui-root-repair.mjs`를 추가하고 validate에 연결했습니다.
- `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge, README를 `2.0.81`로 동기화했습니다.

## v2.0.80 변경사항

- 열린 메뉴 페이지의 상단 HUD와 본문 카드가 다시 겹치지 않도록 `v2080` 메뉴 페이지 구조/클래스 정규화 레이어를 추가했습니다.
- 가방/지도/퀘스트/상점/장비/도감/랭킹의 `runtime-content`를 `v2080-scroll-stack` 세로 스택으로 고정해 본문 카드가 좁은 칸에 눌려 내용이 잘리는 문제를 줄였습니다.
- 카드 목록의 그리드 폭, 카드 내부 컬럼, 버튼 줄바꿈, 설명 텍스트 표시 방식을 재정리해 실 내용보다 칸이 좁게 표현되던 메뉴들을 보강했습니다.
- 특히 가방 슬롯/원장/자동 판매 루프와 퀘스트/상점 긴 목록이 본문 카드 안에서 세로 드래그 스크롤되도록 스크롤 셀렉터와 `touch-action: pan-y` 범위를 확장했습니다.
- 메뉴 닫기 `×` 버튼은 스크롤 카드보다 위에 남기되, 드래그 스크롤 후 버튼 클릭이 막히지 않도록 기존 닫기 훅을 유지했습니다.
- 개척바는 더 짧은 HUD 보조 바로 유지하고, 건물 정보/캐릭터 정보/건설 UI보다 위에 올라오지 않도록 계층을 낮췄습니다.
- 마을 타일 클릭 좌표는 canvas CSS 크기와 Pixi 내부 screen 크기 차이를 보정하고, 클릭 지점 주변 다이아몬드 타일 중 실제 중심에 가장 가까운 타일로 스냅하도록 `tileFromPointer`를 보강했습니다.
- 캐릭터 방향 고정 로직은 변경하지 않았고, `src/villageWorld.ts` 변경 범위는 타일 클릭 좌표 보정과 v2080 감사 마커 추가에 한정했습니다.
- `v2.0.80` 전용 검증 스크립트 `check-v2080-menu-page-grid-tile-hitbox-audit.mjs`를 추가하고 validate에 연결했습니다.
- `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge, README를 `2.0.80`으로 동기화했습니다.

## GitHub Actions / npm registry 주의

`package-lock.json`에 아래 문자열이 들어가면 안 됩니다.

```text
packages.applied-caas
applied-caas-gateway
10.192.
internal.api.openai
```

현재 작업 컨테이너에서는 public npm registry DNS 조회가 `EAI_AGAIN`으로 실패할 수 있습니다. 이 경우 `typecheck`와 `build`는 의존성 미설치로 실패하지만, lockfile 오염과는 별개입니다.
