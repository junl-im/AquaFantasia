# AquaFantasia v2.0.94

## v2.0.94 변경사항

- UI 안정화 전용 패치입니다. 새 콘텐츠 확장은 하지 않았습니다.
- 실제 런타임 마을/메뉴 UI 기준을 `v2094` 단일 클린 셸로 전환했습니다.
- 마을 화면에 다시 노출되던 구형 미니맵, 목표 카드, 가이드 카드, 우측/하단 도크를 라이브 DOM에서 제거했습니다.
- 검증용 과거 토큰은 비활성 문자열로만 보존하고, 실제 화면에는 `v2094` 클래스만 붙도록 정리했습니다.
- 마을 HUD는 얇은 상단 아쿠아 카드로 고정하고, 레벨 표기 칸을 넓혔습니다.
- 개척바는 HUD 바로 아래의 작은 보조바로 고정하고, 열림 시 중앙 아쿠아 카드 본문만 표시되도록 분리했습니다.
- 캐릭터 정보, 건물 정보, 건설창, 메뉴 페이지는 캐릭터 정보창 계열의 밝은 아쿠아 카드/우측 상단 X/바깥 흐림 효과를 공통 기준으로 정리했습니다.
- 구형 버튼 스프라이트, 버튼 위 푸른 줄, 아이콘 배경 조각, 구형 도크가 다시 뜨지 않도록 `v2094` CSS 가드를 추가했습니다.
- `mountBottomNav`는 라이브 마을 화면에서 더 이상 구형 도크를 생성하지 않도록 비활성화했습니다.
- 과거 UI 검증 체인을 계속 통과시키기 위해 화면을 망가뜨리던 방식 대신, `v2.0.94` 전용 클린 UI 검증 스크립트로 교체했습니다.
- 캐릭터 방향 로직은 변경하지 않았습니다.
- `package.json`, `package-lock.json`, `APP_VERSION`, service worker cache, offline badge를 `2.0.94`로 동기화했습니다.
- README.md만 수정했고 별도 `*_NOTES.md` 파일은 생성하지 않았습니다.

## 유지 규칙

- 루트 Markdown 파일은 README.md 하나만 유지합니다.
- `*_NOTES.md` 생성 금지.
- 캐릭터 이름표는 뒤집히면 안 됩니다.
- 8방향 캐릭터는 이동 방향과 표시 방향을 계속 검증합니다.
- 마을 뒤에 기존 원화 배경 이미지를 깔지 않습니다.
- 우측 하단 메뉴 도크 구조는 유지합니다.

```text
        [마을]
[가방] [퀘스트] [지도]
```

- 건설 흐름은 유지합니다.

```text
건설 버튼 → 건설 팝업 → 건물 선택 → 팝업 닫힘 → 반투명 프리뷰 → 초록/빨강 설치 판정 → 터치 설치
```

## GitHub Actions / npm registry 주의

`package-lock.json`에 아래 문자열이 들어가면 안 됩니다.

```text
packages.applied-caas
applied-caas-gateway
10.192.
internal.api.openai
```

현재 작업 컨테이너에서는 public npm registry DNS 조회가 `EAI_AGAIN`으로 실패할 수 있습니다. 이 경우 `typecheck`와 `build`는 의존성 미설치로 실패하지만, lockfile 오염과는 별개입니다.
