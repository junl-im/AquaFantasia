# v8.4.0 UI 점검 보고서

## 확인한 문제
- v8.3.0의 9:16 구성도 surface 방식은 비율은 안정적이지만 UI가 실제 게임보다 작아 보이는 문제가 있었다.
- 낚시 화면에 장비/미끼/오늘 목표 카드가 남아 있어 플레이 면적이 줄어들었다.
- 하단 메뉴가 구성도 이미지와 투명 hotspot에 의존해서 배경이 어울리지 않는 느낌이 있었다.
- 랭킹 구성도에는 테스트용 가짜 유저가 구워진 상태라 실제 사용자 기준 정보와 맞지 않았다.
- 캐릭터 스프라이트가 작은 원본을 크게 확대해서 흐려질 수 있었다.

## 처리 결과
- 낚시 화면은 전체 화면 PixiJS 스테이지로 분리했다.
- 낚시 배경은 v84 고해상도 WebP 배경으로 교체했다.
- 캐릭터는 4x 보정 PNG로 교체했다.
- 하단 네비는 실제 아이콘 버튼형 UI로 다시 표시한다.
- 랭킹 화면은 가짜 유저를 제거하고 내 기록 한 줄만 보여준다.
- 카카오/인앱 브라우저 몰입형 재시도 훅을 추가했다.

## 위험/주의
- v13 전체 구성도 이미지를 쓰는 메뉴 화면은 아직 완전한 런타임 UI가 아니다.
- 우측 상단 버튼 제거는 구성도 원본을 완전히 재작업한 것이 아니라 오버레이 클리너 방식이다.
- 실제 서버 랭킹은 Firestore 또는 Realtime Database 설계와 규칙 배포가 필요하다.
