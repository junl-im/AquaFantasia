# v8.3.0 UI Polish Audit

## 점검 결론

빌드/타입 검사는 통과했지만, v8.2.0 UI에는 시각 품질 문제 가능성이 있었다. 가장 큰 문제는 v13 전체 구성도 이미지를 화면 비율에 맞춰 직접 늘리는 구조였다. v8.3.0에서는 원본 디자인 표면을 유지하고 화면에는 카메라처럼 crop해서 보여주는 방식으로 정리했다.

## 체크리스트

| 항목 | v8.2.0 상태 | v8.3.0 조치 |
|---|---|---|
| v13 화면 비율 | 기기 비율에 따라 직접 늘어남 | 1080x1920 design surface 유지 |
| 시작 화면 비율 | 1024x1536 아트와 hitbox 좌표 분리 가능 | start design surface 안에 이미지와 버튼 배치 |
| 로그인 유지 버튼 | 존재하지만 crop 기기에서 위치 불안정 가능 | 실제 버튼형 UI 유지 + surface 좌표 적용 |
| 하단 탭 | 8탭 transparent nav 유지 | v13 surface 폭 기준 정렬 |
| 낚시 stage | root 기준 위치 | v13 surface 내부 기준 위치 |
| 릴 패널/콤보 | root 기준 위치 | v13 surface 내부 기준 위치 |
| 터치 타겟 | 일부 CTA 44px 미만 가능 | 검사 스크립트로 최소 크기 확인 |
| 검증 스크립트 | v8.2.0 전용 | v8.3.0 UI polish 검사 추가 |

## 자동 검증

- v13 탭 구성도 8개가 모두 1080x1920인지 확인
- 시작 화면이 1024x1536인지 확인
- 주요 코드 토큰이 존재하는지 확인
- 모든 v13 hotspot이 390x844 기준 44px 이상인지 확인
- package/data/service worker 버전과 cache name 일치 여부 확인

## 수동 확인 필요

자동 검증은 좌표와 빌드 안정성만 확인한다. 실제 미감은 모바일 기기에서 직접 봐야 한다. 특히 다음은 수동 확인이 필요하다.

- 시작 화면에서 `익명 서버연동`과 `이 기기에서 로그인 유지` 간격
- v13 구성도 하단 탭 그림과 실제 터치 반응 위치
- 낚시 stage가 구성도 패널과 자연스럽게 섞이는지
- 긴 기기에서 좌우 crop이 너무 과하지 않은지
