# AquaFantasia v8.4.0 최종 정리 통파일

## 패치명
**v8.4.0 Fullscreen Fishing UI Polish**

## 이번 패치의 기준
사용자가 지적한 핵심 문제를 우선 반영했다.

- 전체 UI가 작아진 문제
- 최하단 메뉴 버튼 배경이 어울리지 않는 문제
- 낚시터가 메뉴 구성도 안에 갇혀 플레이 맛이 줄어든 문제
- 낚싯대 / 미끼 / 오늘 목표 카드가 낚시 화면에서 불필요하게 공간을 차지하는 문제
- 최근 포획란 위치가 너무 위에 있는 문제
- 랭킹에 가짜 유저가 표시되는 문제
- 우측 상단 불필요 버튼 2개가 메뉴마다 따라다니는 문제
- 캐릭터가 흐릿해 보이는 문제
- 카카오 브라우저에서 풀화면/몰입형 표시가 풀린 뒤 복구되지 않는 문제
- 세로 고정은 절대 유지해야 하는 조건

## 반영 요약

### 1. 낚시 화면 구조 변경
v13 전체 구성도 이미지를 낚시 화면에 계속 쓰지 않도록 변경했다. 낚시 화면은 이제 다음 구조다.

```text
전체 화면 PixiJS 낚시 배경
상단 소형 HUD
전체 화면 터치 낚시 스테이지
낚시 시작 CTA
최근 포획 패널
하단 탭 네비
```

제거된 낚시 화면 요소:

- 낚싯대 카드
- 미끼 카드
- 오늘 목표 카드
- 해당 카드로 이동하던 v13 투명 hotspot

유지한 요소:

- 낚시 시작 버튼
- 릴링 패널
- 장력 게이지
- 최근 포획
- 하단 탭 네비
- 세로 고정

### 2. 낚시 배경 품질 개선
기존 `v12/bg/*_portrait.webp`는 일부 배경이 패널처럼 잘린 느낌과 블러가 강했다. v8.4.0에서는 사용자가 제공한 4x 업스케일 배경을 다시 보정해 `public/assets/v84/bg`에 넣었다.

추가된 배경:

- `ocean_full.webp`
- `lake_full.webp`
- `pier_full.webp`
- `river_full.webp`
- `stream_full.webp`
- `deepsea_full.webp`

`src/data.ts`의 수역 배경은 v84 배경으로 연결했다.

### 3. 캐릭터 선명도 보정
기존 캐릭터 원본은 작아서 PixiJS에서 확대될 때 흐려질 수 있었다. v8.4.0에서는 `chibi_fisher_01.png`를 4x 리샘플링 후 색감/대비/샤픈 보정해 다음 파일로 연결했다.

- `public/assets/v84/characters/chibi_fisher_01_hd.png`

### 4. 하단 네비 개선
v8.3.0의 v13 투명 hotspot 네비는 구성도 이미지에 너무 의존했다. v8.4.0에서는 실제 아이콘과 텍스트가 보이는 버튼형 하단 네비로 복구했다.

- 8개 탭 유지: 마을 / 낚시 / 장비 / 가방 / 도감 / 상점 / 미션 / 랭킹
- 고정 위치 유지
- 활성 탭은 금색 버튼형 표시
- 이상한 배경 이미지는 제거하고 CSS 기반 2.5D 버튼 느낌으로 정리

### 5. 메뉴 화면 크기 보정
장비/상점/미션/가방/도감/마을은 아직 v13 구성도 이미지를 사용하지만, UI가 너무 작아 보이지 않도록 v13 design surface를 106.5%로 확대했다.

### 6. 우측 상단 불필요 버튼 정리
v13 구성도에 구워진 우측 상단 버튼 2개는 실제 기능이 붙어 있지 않아 혼란을 만든다. v8.4.0에서는 `v13-top-action-cleaner`로 해당 영역을 시각적으로 덮어 제거했다.

### 7. 랭킹 가짜 데이터 제거
랭킹 화면은 v13 구운 이미지 대신 실제 저장 데이터 기반 화면으로 변경했다.

현재 표시:

- 나 #1
- 로컬 테스트 계정 또는 익명 서버 연동 계정 상태
- 최고 콤보
- 성공 횟수
- 누적 포획
- 골드

가짜 유저 여러 명은 제거했다.

### 8. 카카오/인앱 브라우저 몰입형 재시도
다음 상황에서 세로 몰입형 화면을 다시 정렬/요청한다.

- 메뉴 클릭
- 낚시 시작
- 릴링 액션
- pointerdown/touchend
- visibilitychange/pageshow

브라우저가 Fullscreen API를 거부해도 CSS portrait cage는 계속 유지된다.

## 이번 패치에서 솔직히 남은 부분
장비/상점/미션/가방/도감/마을은 아직 v13 전체 구성도 기반이다. 보이는 안정성은 개선했지만 완전한 런타임 UI는 아니다. 다음 단계에서는 v13 이미지를 더 이상 통짜 배경으로 쓰지 않고, 사용자가 제공한 프레임/버튼/아이콘을 9-slice 스타일로 분해해서 실제 DOM/Pixi UI로 재구성해야 한다.
