# v6.7.1 Kakao Orientation Safe Audit

## Problem
KakaoTalk in-app browser can ignore PWA manifest orientation and may jump orientation when fullscreen/orientation APIs are called.

## Fix
- Added hostile in-app browser detection.
- Disabled requestFullscreen and screen.orientation.lock inside Kakao/in-app shells.
- Kept silent CSS portrait cage fallback.
- Preserved portrait-primary manifest for PWA/external browsers.
