# v5.5.5 Auto Cache Sweep Audit

## 발견 문제

- v5.5.2/v5.5.4 복구 카드가 오류 감지 후 `open` 상태로 남아 모바일 화면을 가릴 수 있었다.
- 캐시 정리 안내가 버튼 기반이라 사용자가 직접 눌러야 복구되는 흐름이었다.
- Service Worker keep prefix가 v5.5.2로 남아 있어 최신 v5.5.5 캐시와 구버전 캐시가 섞일 가능성이 있었다.

## 수정

- v5.5.5 자동 캐시 정리 런타임을 head 최상단 런타임으로 추가했다.
- 배포 빌드 버전이 바뀌면 이전 CacheStorage를 자동 삭제하고 필요할 때만 1회 reload한다.
- 정상 업데이트/일반 정리에서는 복구 팝업을 띄우지 않거나 즉시 숨긴다.
- v5.5.2/v5.5.4의 레거시 복구 카드 DOM을 주기적으로 숨겨 sticky overlay를 방지한다.
- `AQUA_FORCE_CACHE_SWEEP` Service Worker 메시지를 추가해 클라이언트와 SW 양쪽 캐시 정리를 맞췄다.

## 검증

- `npm run validate`
- `npm run runtime:check`
- `npm run renderer:check`
- `npm run audit`
- `npm run atlas:check`
- `npm run clean:report`
