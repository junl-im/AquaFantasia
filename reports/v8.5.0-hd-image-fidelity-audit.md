# v8.5.0 HD Image Fidelity Audit

## 진단
v8.4.0 기준으로 화면이 흐릿해 보이는 가장 큰 원인은 다음이었다.

1. v13 전체 화면 구성도 WebP가 1080×1920임에도 파일당 약 120~150KB로 강하게 압축되어 디테일이 뭉개짐.
2. 낚시 배경 텍스처가 세로 화면보다 작아 DPR이 높은 모바일에서 확대 렌더링될 가능성이 큼.
3. PixiJS 캔버스 해상도 상한이 낮아 일부 모바일에서 캐릭터/물고기가 부드럽지만 흐릿하게 보일 수 있음.
4. 일부 투명 캐릭터 시트에 중복 포즈와 잔여 조각이 남아 런타임 단일 캐릭터로 쓰기에 부적합했음.

## 조치
- `public/assets/v85`를 새 런타임 에셋 레이어로 추가.
- v13 구성도는 원본 PNG에서 재생성하여 과압축 방지.
- 낚시 배경은 1440×2560 세로 텍스처로 재생성.
- 캐릭터는 단일 보트 캐릭터로 분리 후 고해상도 PNG로 저장.
- 물고기, 네비 아이콘, 주요 UI 패널/버튼을 v85 경로로 연결.
- CSS 최종 레이어에서 브라우저 이미지 렌더링 안정화.
- Service Worker 캐시 버전과 precache 대상을 v85로 갱신.

## 한계
이미지 보정은 원본의 실제 디테일을 새로 창조하지는 않는다. 원화급 선명도에 더 가까워지려면 추후 각 탭 화면을 통짜 구성도가 아니라 원본 파츠 기반 UI로 다시 조립해야 한다.
