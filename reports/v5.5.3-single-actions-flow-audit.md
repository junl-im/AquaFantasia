# v5.5.3 Single Actions Flow Audit

## Finding

Duplicate automatic GitHub Actions runs were caused by an overlay patch adding a new static validation workflow while the original Pages deployment workflow remained in the repository.

## Fix

The Pages workflow now performs validation and deployment in one automatic job. The static validation workflow has been converted to manual-only mode to prevent duplicate push runs. Runtime checker scripts now accept the Pages workflow as the canonical validation workflow, so the manual static workflow can be removed from the repository if desired.

## Expected result

On push to `main` or `master`, only this automatic workflow should run:

- Aqua Fantasia Validate & Deploy Pages

The manual workflow may still be visible in the Actions sidebar until the file is deleted from the repository, but it should not run on push.
