# v6.9.0 Start / Back / Bite UI Audit

## 분석 결과
v6.8.0에서 시작 화면은 레퍼런스 아트에 맞게 정리되었으나, 사용자가 쓰지 않는 공지사항/고객센터/설정/하단 메뉴가 시작 전 화면에 남아 있었습니다. 또한 낚시 입질 시 플레이어가 다음 조작을 직관적으로 인지하기 어려웠고, 모바일 웹 특성상 뒤로가기 버튼이 웹페이지 이동처럼 동작했습니다.

## 조치
- 시작 화면 정리본 `start_screen_clean_v690` 생성
- 시작 화면 터치 영역을 출항/새 게임/익명 서버연동/로그인 유지로 축소
- 로그인 유지 체크 UI를 별도 오버레이로 보정
- `showBiteCallout()` 추가
- `installBackNavigationGuard()` 추가
- `showGameConfirm()` 기반 게임형 확인창 추가
- 카카오톡 세로 대응 유지

## 검증 명령
- npm run validate
- npm run runtime:check
- npm run audit
- npm run typecheck
- npm run build
- npm audit --audit-level=high
