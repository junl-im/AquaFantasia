# v5.5.1 Runtime Error Audit

## 진단

정적 문법 검사는 통과했지만, 실제 실행 중에는 다음 충돌 가능성이 있었습니다.

1. `#cast-btn`에 기존 단일 HTML inline handler `onclick="castLine()"`가 남아 있음.
2. v5.3 모듈 런타임이 같은 버튼에 `fishing.startCast()`를 추가로 바인딩함.
3. 결과적으로 CAST 한 번에 레거시 `gameState` 기반 낚시 루프와 새 `aquaStore` 기반 낚시 루프가 동시에 진행될 수 있음.
4. `#bite-target`, `#reel-action-btn`, `#fishing-visual.onclick`도 같은 방식으로 레거시 루프와 새 모듈 루프가 섞일 수 있음.
5. v5.5 이후 Service Worker stale-while-revalidate 캐시가 오래된 런타임 파일을 한 번 더 제공할 수 있어, 패치 직후 오류가 재발할 수 있음.

## 조치

- v5.5.1 핫픽스 런타임에서 inline onclick을 제거하고, v5.3 API 중심으로 CAST/BITE/REEL 입력을 단일화했습니다.
- v5.3 API가 없을 때만 레거시 함수로 fallback하도록 구성했습니다.
- 모든 런타임 모듈 script URL에 `?v=5.5.1`을 붙여 브라우저/SW 캐시 충돌을 줄였습니다.
- Service Worker 버전과 registration 옵션을 갱신했습니다.

## 남은 권장 작업

- 다음 대규모 패치에서는 기존 단일 HTML의 레거시 낚시 루프를 완전히 제거하고, v5.3+ 모듈 시스템만 남기는 구조로 정리하는 것이 좋습니다.
- Firebase CDN 실패 시 게스트 모드가 항상 먼저 뜨도록 로그인 플로우도 별도 모듈화하는 것이 좋습니다.
